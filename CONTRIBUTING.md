# Contributing to Stouffer Labs

Thank you for your interest in contributing! We welcome contributions of all kinds — bug reports, feature requests, documentation improvements, and code.

## Getting Started

1. **Fork** the repository
2. **Clone** your fork locally
3. **Create a branch** for your change (`git checkout -b my-change`)
4. **Make your changes** and test them
5. **Commit** with a clear message describing what you changed and why
6. **Push** to your fork and open a **Pull Request**

## Development Guidelines

- Follow the existing code style in the project
- Write tests for new functionality
- Keep PRs focused — one logical change per PR
- Update documentation if your change affects usage

## Reporting Issues

- Use the issue templates provided (bug report or feature request)
- Include steps to reproduce for bugs
- Provide context about your environment (OS, language version, etc.)

## Code Review

All PRs are reviewed before merging. We aim to review within a few days. Feedback is meant to improve the code, not criticize — we're all here to build good software.
